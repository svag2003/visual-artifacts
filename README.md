# Visual Artifacts

A [Claude Code](https://claude.com/claude-code) skill that turns **any** project, codebase, or idea into one shareable HTML page that *draws* its architecture instead of describing it in a wall of text.

Every artifact has three tabs — **Technical Architecture**, **User Workflow**, **Business Strategy** — each led by a real diagram. The same file serves two readers on one link: a **human** opens it and sees the picture; an **agent** fetches it and reads the architecture straight out of the HTML (the diagrams are [Mermaid](https://mermaid.js.org) *text*, so no OCR or screenshot-parsing needed).

**▶ [See a live example →](https://svag2003.github.io/visual-artifacts/)** — a Visual Artifact *of this skill*, generated by the skill and served straight from this repo. Open it, switch tabs, flip themes, tap a diagram to zoom.

## Install

```text
/plugin marketplace add svag2003/visual-artifacts
/plugin install visual-artifacts@visual-artifacts
```

Then use it:

```text
/visual-artifacts:html-it ./my-project
/visual-artifacts:html-it "a webhook that posts Stripe events to Slack"
```

Or try it without installing:

```bash
claude --plugin-dir ./visual-artifacts
```

## What you get

One `.html` file, saved to `visual-artifacts/`:

- **Self-contained by default** — Mermaid is baked in, so the file works fully offline, on `localhost`, or emailed, and makes **zero** outside calls. Private unless you choose to publish.
- **Diagram-driven** — a real topology graph (Technical), a user-journey sequence (User Workflow), and a value/positioning view (Business), plus a consistent component kit (status boards, stat cards, funnels).
- **Agent-readable** — predictable structure (`<section data-lens="…">`, `<pre class="mermaid">`) so any agent can parse it.

## Modes

| Command | Result |
|---|---|
| `python3 references/build.py artifact.html` | **Self-contained** (~3.3 MB, offline, private) — the default |
| `python3 references/build.py artifact.html --lightweight` | **Lightweight** (a few KB, loads Mermaid from a CDN) — for publishing many artifacts to one host |

## Share it

The output is static — host it anywhere and send the link; the same URL renders for a human and reads cleanly for an agent.

- **GitHub Pages** — commit the `.html`, enable Pages, share the URL (free public link).
- **Any static host** — S3 + CloudFront, Netlify, Vercel, or a plain web server.
- **Keep it private** — just don't publish; open the file or serve it on `localhost`.

## Modify it

It's a normal skill — fork the repo and edit `SKILL.md` (the instructions), `references/template.html` (the look + component kit), or `references/build.py` (the assembler). Re-vendor Mermaid any time with:

```bash
curl -fsSL https://cdn.jsdelivr.net/npm/mermaid@11/dist/mermaid.min.js -o references/mermaid.min.js
```
